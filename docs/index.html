01 Premise

Payload delivery server for authorized red team engagements with secure access control.

Deploy a single Go binary that serves payloads via 12 different loader methods with dual TLS listeners and comprehensive anti-forensics measures. Token-gated access with HMAC-SHA256 validation.

Server implements AES-256-GCM encryption at rest, configurable TTL and IP constraints, memory zeroing on shutdown, and nginx decoy infrastructure. All operations use Go standard library with zero external dependencies.

02 Specs

What's in the server.

DELIVERY
12 loader methods — socat TLS/TCP, curl, wget, bash /dev/tcp, python, memfd fileless.
SECURITY
Token-gated access with HMAC-SHA256, TTL, IP locks, max-use limits, timing attack protection.
CRYPTO
AES-256-GCM encryption at rest with memory zeroing, runtime.KeepAlive prevents elision.
LISTENERS
Dual TLS servers — HTTPS :443 and raw TLS :4443 with TLS 1.3 minimum.
STEALTH
Nginx decoy infrastructure with identical 404 pages, server headers, indistinguishable responses.
STACK
Single Go binary · zero external deps · interactive console · Catppuccin Mocha palette.
03 Quickstart

Build, certificate, serve, operate.

Server deployment: static binary → TLS certificate generation → dual listeners → interactive operator console with payload management.

# Build static binary (Go 1.21+ required)
$ git clone https://github.com/Real-Fruit-Snacks/Wellspring
$ cd Wellspring && make build

# Generate TLS certificate with SNI decoy
$ ./build/wellspring -gencert -sni cloudflare-dns.com

# Start token-gated payload server (dual TLS listeners)
$ ./build/wellspring -cert server.crt -key server.key

# Interactive console operations (payload management)
# payload add ./implant --name undertow
# generate p1 all --host 10.0.0.1 --port 443
04 Delivery

Twelve loader methods for diverse environments.

Methods range from HTTPS pull-and-exec to fileless execution via memfd_create. Raw TLS listener supports socat connections, HTTPS listener serves HTTP-based loaders with token validation.

SOCAT
socat-tls, socat-tcp, socat-memfd, socat-pty
HTTP
curl, wget, python urllib, perl IO::Socket::SSL
SHELL
bash /dev/tcp, netcat raw pull
FILELESS
memfd_create, /dev/shm tmpfs staging
05 Architecture

Three-layer design with interactive console.

HTTP/TLS server layer manages dual listeners, payload manager handles AES-256-GCM encryption at rest, interactive CLI provides operator interface with token generation and revocation.

cmd/wellspring/
main.go                 # entry, flags, init
internal/
  server/               # dual TLS listeners
  payload/manager.go    # AES-256-GCM at rest
  loader/registry.go    # 12 delivery methods
  cli/cli.go            # interactive console
06 Authorization

Authorized red team engagements with explicit permission.

Server designed for payload delivery in scoped, written, and signed engagements. Generates and delivers payloads that should only be used against systems you own or have proper authorization to test.

Report vulnerabilities via private security advisories, never public issues. Server includes no command and control, exploitation, persistence, or data exfiltration capabilities.

Payload delivery only — 12 loader methods with token authentication, TTL constraints, and anti-forensics measures.
